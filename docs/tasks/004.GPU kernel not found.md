Current failures are:

CUDA:
nvcc fatal : Cannot find compiler 'cl.exe' in PATH

and:

OptiX:
Unable to compile OptiX kernels at runtime.
Set OPTIX_ROOT_DIR environment variable to a directory containing the OptiX SDK.

Root cause: the Windows install/acycles.exe has no bundled GPU kernels, so at preview runtime it tries to compile CUDA/OptiX kernels. That works only in a
Visual Studio developer environment with CUDA and OptiX configured, but the Studio server runtime does not have that.

I changed the CUDA/OptiX configure scripts to build and install GPU kernels at build time:

- scripts/configure-local-cuda-optix.ps1:153
- scripts/configure-local-cuda-optix.sh:24

Added:

WITH_CYCLES_CUDA_BINARIES=ON
WITH_CYCLES_CUDA_BUILD_SERIAL=ON
CYCLES_CUDA_BINARIES_ARCH=sm_86

Also the Windows script now passes:

CYCLES_RUNTIME_OPTIX_ROOT_DIR=$OptixRoot

You need to reconfigure, not only rebuild. On Windows, run from F:\GitRepo\acycles with your real Windows OptiX SDK path, for example:

.\scripts\configure-local-cuda-optix.ps1 -OptixRoot "C:\ProgramData\NVIDIA Corporation\OptiX SDK 8.0.0"
.\scripts\build-local.ps1 -Target install -Config Release

Then check that these files exist under F:\GitRepo\acycles\install\lib:

kernel_sm_86.cubin.zst
kernel_optix.ptx.zst
kernel_optix_shader_raytrace.ptx.zst

Without those files, CUDA/OptiX preview will keep trying runtime compilation and fail in Studio.
